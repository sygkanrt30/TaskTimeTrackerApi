# Task Time Tracker API

Сервис для учёта рабочего времени сотрудников. Позволяет создавать задачи, управлять их статусами и фиксировать временные отрезки, затраченные на выполнение.

## Функционал

- **Управление задачами (Task):** создание, просмотр по ID, изменение статуса (`NEW`, `IN_PROGRESS`, `DONE`).
- **Учёт времени (TimeRecord):** создание записи о затраченном времени на задачу (только для задач со статусом `DONE`), получение суммарного времени работы сотрудника за определённый период.
- **Аутентификация и безопасность:** JWT-аутентификация (токен хранится в HttpOnly cookie), регистрация и вход в систему.
- **Документация API:** интерактивная документация Swagger UI.

## Используемые технологии

- **Java 21**
- **Spring Boot 3.5.9**
- **Spring Security**, **Nimbus JOSE JWT**
- **MyBatis**, **PostgreSQL**, **Flyway**
- **Maven**, **Lombok**, **MapStruct**
- **SpringDoc OpenAPI** (Swagger)
- **Docker**, **Google Jib**

## Инструкция по запуску (для пользователя)

### Быстрый запуск через Docker

Этот способ не требует установки Java, Maven или PostgreSQL на вашем компьютере. Нужен только **Docker Desktop**.

#### Предварительные требования
- Установленный **Docker Desktop** (для Windows/Mac) или **Docker Engine** (для Linux)
- Свободные порты: `8080` (приложение) и `5435` (база данных)

#### Шаги по запуску

1.  **Создайте файл `.env`** в любой пустой папке (например, `task-tracker`) со следующим содержимым. Это нужно для настройки ключа шифрования JWT:
    ```dotenv
    JWT_COOKIE_TOKEN_KEY=ваш-супер-секретный-ключ-минимум-из-32-символов
2. Создайте файл docker-compose.yml из проекта в ту же папку

3. Запустите приложение командой в терминале из этой папки:
`docker-compose up -d`

Будут скачаны образы PostgreSQL и самого приложения, после чего оба контейнера автоматически запустятся.

#### Проверьте работоспособность

Откройте в браузере Swagger UI: http://localhost:8080/swagger-ui.html

Если интерфейс загрузился — приложение работает.

#### Остановка приложения

Чтобы остановить приложение:
`docker-compose down`

Чтобы полностью удалить и базу данных, добавьте флаг -v: 
`docker-compose down -v`

## Тестирование API (Postman)
Для удобного тестирования всех эндпоинтов прилагается готовая коллекция Postman.

#### Импорт коллекции

1. Скачайте файл коллекции: TaskTimeTracker.postman_collection.json в репозитории.

2. В приложении Postman нажмите кнопку Import и выберите скачанный файл.

### Структура запросов в коллекции

Коллекция содержит все необходимые эндпоинты:

| Папка | Запрос | Метод | Эндпоинт |
|-------|--------|-------|----------|
| **Auth** | Register | POST | `/api/v1/auth/reg` |
| **Auth** | Login | POST | `/api/v1/auth/login` |
| **Task** | Create Task | POST | `/api/v1/task` |
| **Task** | Get Task by ID | GET | `/api/v1/task/{id}` |
| **Task** | Update Status | PATCH | `/api/v1/task/update/status` |
| **TimeRecord** | Create Time Record | POST | `/api/v1/task/time_record` |
| **TimeRecord** | Get Work Time | GET | `/api/v1/task/time_record/time_of_work/for_period` |

> **Примечание:** Запросы с `X-XSRF-TOKEN` в заголовках будут работать только если вы предварительно получили актуальный CSRF-токен.

## Дополнительная информация

### JWT ключ 
Секретный ключ, указанный в файле .env, должен содержать не менее 32 символов. Это требование библиотеки шифрования.

### CSRF защита
В проекте настроена CSRF-защита через cookie. Для не-GET запросов из браузера может потребоваться CSRF-токен. В коллекции Postman это автоматически решается через промежуточный запрос GET /api/v1/auth/csrf (если он реализован) или путём чтения cookie из ответа после логина.
